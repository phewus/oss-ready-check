# Check Reference

OSS Ready Check looks for common files and folders that make a repository easier to use and contribute to.

## README

A README should answer:

- What does this project do?
- Who is it for?
- How do I install or run it?
- What is a minimal example?
- How can I contribute?

## License

A license tells users what they are allowed to do with the code. Without a license, many people and companies cannot safely use the project.

## Contributing guide

A contributing guide helps people submit better issues and pull requests. It should include setup steps, tests, style notes, and review expectations.

## Security policy

A security policy tells users how to responsibly report vulnerabilities without exposing everyone to risk.

## Tests

Tests help maintainers accept contributions with more confidence.

## Examples

Examples reduce confusion and help users understand the project quickly.
